Documentação API

Esta documentação descreve os endpoints e o seu retorno na API da Loja Online. Algumas das rotas são protegidas, por isso é necessário autenticar-se com um token de acesso.
Para informações legais, consulte os Termos de Serviço e a Política de Privacidade.

Autenticação

Para aceder aos endpoints protegidos, é necessário autenticar-se com um token de acesso.


POST /accounts/login

Autentica um utilizador e devolve um token de acesso e o id da conta utilizada.

A palavra passe é encriptada utilizando a biblioteca brypt. O token enviado tem validade de 30 dias.

Body

{ 
  "email": "exemplo@email.com",
  "password": "password123"
}

Resposta

Se os dados tiverem corretos este endepoint retorna o objeto abaixo:

{
  "token": "eyJhb...yMzUw",
  "accID": "60f...c3"
}

Se os dados estiverem incorretos, o endpoint retorna um erro 401 e uma mensagem:

{ message: 'O dados de início de sessão estão incorretos!' }

GET /accounts/token

Devolve o accounts da conta através do token de acesso.

*Este endpoint necessita de header com o token de acesso.

Resposta

Se a conta existir, o endpoint retorna o objeto com a conta:

{
  "token": "eyJhb...",
  "accID": "60f...c3"
}

Se o token não existir, o endpoint retorna um erro 401 e uma mensagem:

{ message: 'O token não foi enviado!' }

Se o JWT Secret não existir, o endpoint retorna um erro 500 e uma mensagem:

{ message: 'O JWT secret não foi enviado!' }

Se o token for inválido, o endpoint retorna um erro 403 e uma mensagem:

{ message: 'Existiu um erro na verificação do token:' + error }

POST /accounts/reset-pw

Envia um email para redifinição de palavra passe.

Body

{
  "email": "exemplo@email.com"
}

Resposta

Se o email for enviado, o endpoint retorna a mensagem abaixo:

{ message: 'Email enviado com sucesso!' }

Se o email do cliente não existir, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Conta não encontrada!' }

Pagamentos

Endpoints para processar pagamentos com Stripe.


GET /stripe/retrieve-payment-id

Este endpoint é utilizado para obter o paymentId para o pagamento.

*Este endpoint necessita de header com o token de acesso.

Resposta

Se existir sessionId, o endpoint retorna o objeto abaixo:

{
  "paymentId": "pi_1J...3"
}

Caso contrário, o endpoint retorna um erro 400 e uma mensagem:

{ error: 'Não existe session ID' }

POST /stripe/create-checkout-session

Este endpoint é utilizado para criar um pagamento com Stripe.

*Este endpoint necessita de header com o token de acesso.

Body

{
  "amount": 1000,
  "orderInfo": {
    "contactInfo": "exemplo@email.com",
    "firstName": "Primeiro",
    "lastName": "Último",
    "street": "Rua do Exemplo",
    "addressExtra": "Ap 1",
    "postalCode": "1234-567",
    "city": "Exemplo",
    "country": "Portugal"
    "nif": "123456789" *Campo Opcional
  }
}

Resposta

Se a sessão for criada, o endpoint retorna o objeto abaixo:

{
  "session": {
    "payment_method_types": ["card"],
    "line_items": [
      {
        "price_data": {
          "currency": "eur",
          "product_data": {
            "name": "Produto",
          },
          "unit_amount": 1000
        },
        "quantity": 1
      }
    ],
    "mode": "payment",
    "success_url": "${domain}/successful-payment?hash=4389236587543",
    "cancel_url": "${domain}/unsuccessful-payment",
    "expand": ["payment_intent"],
  }
}

Se a sessão não for criada, o endpoint retorna um erro 500 e uma mensagem:

{ error: 'Ocorreu um erro ao criar a sessão de pagamento!' }

POST /products

Adiciona um novo produto.

*Este endpoint necessita de header com o token de acesso.

Body

{
  "reference": "RTG4",
  "name": "Produto",
  "description": "Descrição do produto",
  "price": 1000,
  "size": "M",
  "stock": 10,
  "category": "Calças"
}

Resposta

Se o produto for adicionado, o endpoint retorna a mensagem abaixo:

{ message: 'Produto adicionado com sucesso!' }

Se o produto já existir, o endpoint retorna um erro 409 e uma mensagem:

{ message: `Product with reference '${reference}' and size '${size}' already exists`}

Se algum dado for introduzido incorretamente, o endpoint retorna um erro 409 e uma mensagem:

{ message: 'Categoria não encontrada!' }

ou

{ message: 'Tamanho inválido. Os tamanhos permitidos são XS, S, M, L e XL'}

    "reference": "RTG4",

Resposta

{ message: 'Existem campos não preenchidos!' }

Se o produto já existir, o endpoint atualiza a quantidade e envia uma mensagem:

{ message: 'Quantidade do produto atualizada!' }

Para cada produto adicionado, o endpoint retorna a mensagem abaixo:

{ message: 'Produto adicionado ao carrinho com sucesso!' }

POST /orders

Regista uma nova encomenda.

*Este endpoint necessita de header com o token de acesso.

Body

{
  "paymentId": "123456789",
}

Resposta

Se a encomenda for adicionada, o endpoint retorna o objeto order e a mensagem abaixo:

{ order: order, message: 'Encomenda registada com sucesso!' }

Se não existir paymentID, o endpoint retorna um erro 400 e uma mensagem:

{ message: 'O ID do pagamento é obrigatório!' }

Se a conta não existir, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Conta não encontrada!' }

Se não existirem produtos no carrinho, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Carrinho vazio!' }

Se algum campo tiver vazio, o endpoint retorna um erro 400 e uma mensagem:

{ message: 'Todos os campos devem estar preenchidos!' }

Se o produto de uma encomenda não existir, o endpoint retorna um erro 409 e uma mensagem:

{ message: 'O produto com a referência '${reference}' e tamanho '${size}' não existe!' }

Se o produto de uma encomenda não tiver stock suficiente, o endpoint retorna um erro 409 e uma mensagem:

{ message: 'O produto com a referência '${reference}' e tamanho '${size}' não tem stock suficiente!' }

POST /orders/upload-pdf/:id

Cria o PDF de uma encomenda.

*Este endpoint necessita de header com o token de acesso.

File

Envie o PDF no formato multipart/form-data.

Resposta

Se o PDF for criado, o endpoint retorna o URL do pdf:

{ pdfUrl: 'https://exemplo.com/fatura.pdf' }'

Se a encomenda não existir, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Encomenda não encontrada!' }

Se o pdf não for enviado, o endpoint retorna um erro 400 e uma mensagem:

{ message: 'Erro ao ler o ficheiro.' }

*Este endpoint necessita de header com o token de acesso.

Body

  "accountId": "60f...c3",
  "productsIds": ["60f...c3", "60f...c3"],
  "paymentId": "123456789",
  "status": "Pago",
}

Resposta

{ message: 'Encomenda atualizada com sucesso!' }

Se a encomenda não existir, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Encomenda não encontrada!' }

PUT /orders/status/:id

Atualiza o estado de uma encomenda e envia um email ao cliente.

*Este endpoint necessita de header com o token de acesso.

Body

O status pode ser apenas um dos seguintes:

['Em processo', 'Pago', 'Enviado', 'Entregue', 'Cancelado'] 
{
  "status": "Pago",
}

Resposta

Se o estado da encomenda for atualizado, o endpoint retorna a mensagem abaixo:

{ message: 'Estado da encomenda atualizado com sucesso!' }

Se a encomenda não existir, o endpoint retorna um erro 404 e uma mensagem:

{ message: 'Encomenda não encontrada!' }

Se o estado enviado for inválido, o endpoint retorna um erro 400 e uma mensagem:

{ message: 'Estado inválido!' }
